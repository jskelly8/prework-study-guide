Prework Study Guide

HTML

  • Hypertext Markup Language (HTML) - Gives content structure. It's made up of elements which work together to provide content and the basic framework for all pages on the web.
  • Web Application - App accessed via a web browser that allows users to perform tasks on the internet by entering URL.
  • Uniform Resource Locator (URL) - Fundamental web development concept, of which users enter it into a browser, or by clicking a link, to access a web app. It is a unique address given to a specific resource on the web.
  • Element - A type of content on a webpage. Can be seen on webpage & also can be used to divide the content on the page and describe the content to the browser and other developers. Refers to the entire "element" from opening tag to closing tag. (Most common types of elements are: Text, Image, Header, & List & Most common divider type elements are: Meta-related, content division, & Semantic)
  • HTML Attribute - Defines additional info about an element & is always defined in the beginning of an element, never closing the element.
  • Metadata - Data about our webpage that the browser uses to efficiently serve our web users. Typically is not seen by users, but contains important info about accessibility, search engines, & performance.
  • Search Engine Optimization (SEO) - Process of improving the ranking of a website in search engines. Good SEO results in your website to show up in searches more frequently.
  • Child Element - Elements that are nested inside other elements.
  • Parent Element - Element that holds all the nested "children" element(s).
  • Semantic HTML Element - Name of the element that describes it's content and the role it plays on a webpage. Makes your HTML easier to read by the browser and other developers. Makes sure the webpage is accessible to wide variety of users who use assistive technology, like screen readers.
  • Tag - Refers only to what appears inside the angle brackets.
  • Opening Tag & Closing Tag - Tags with content in between opening & closing that are used to make the browser understand how to treat the content. Closing tags always begin with a forward slash.
  • Headings - Larger or more prominent titles & subtitles to organize a page. Elements h1, h2, h3, h4, h5, & h6 represent the level of heading, from largest to smallest (remember to always use in order, and only use h1 once in a webpage).
  • Anchor (a) - Creates a link to the same webpage, or other webpages.
  • Head Element - Contains information, or metadata, about the webpage which helps the browser to render the page correctly. These elements are not visible on the webpage, except for the title element. Is also the "container" for metadata.
  • Body Element - Represents the visible content shown to the user & all the elements that we want the user to see and interact with.
  • Style Element - Defines CSS styles associated with a website.
  • Base Element - Defines the base URL for a website.
  • Link Element - Connects an external resource to the HTML document.
  • Meta Element - Defines metadata such as the character set, description, keywords, author, & viewport.
  • Script Element - Embeds client-side scripts in the HTML document.
  • !DOCTYPE - Declaration that tells the browser that the document is written with the stated language following DOCTYPE.
  • Title Element - Defines the title of the webpage
  • Footer Element - Appears at the bottom of the page & usually contains the author, copyright, contact, sitemap, & navigation.
  • Section Element - Semantic element that defines a section in a document that contains thematic content. Allows us to elegantly organize the content on a webpage.
  • SRC Attribute - Defines the location, or source, of an image file.
  • ALT Attribute - Contains a text string that describes the appearance & functionality of an image. Alt text is important for screen readers and assistive technology & can also improve search engine rankings for your website.

CSS

  • Cascading Style Sheets (CSS) - A programming language used by developers to define how the webpage content is presented to users & give the content style. Defines the colors, aesthetics, visual layout, and organization. Used to define style rules for specific elements of an HTML page and format them using a wide range of style options.

Git

  • Git - Connected to GitHub & accessed via command line, it is software on your computer that tracks changes that you make to your code. Is a version control system that lets you manage and keep track of each version of your project.
  • GitHub - Cloud based hosting service that allows developers to store, manage, and easy collaboration on code. Also allows us to keep track of our projects inside repos that are stored remotely.
  • Command Line Interface (CLI) - A command-line program that allows a user to input text commands to execute operating system functions.
  • Bash - Popular version of a shell that will allow us to perform many tasks from the command line inside the terminal. *Bash & GitBash are the same thing.
  • GitBash - Allows users to use standard Unix commands that otherwise wouldn't be available to a windows user.
  • Terminal - Application that allows us to interact with the shell.
  • Shell - Software that interprets and executes the various commands that we type in the terminal.
  • Secure Shell Key (SSH) - An access credential for the SSH network protocol. Generating a SSH key allows developers to interface with certain remote services without having to constantly retype login info.
  • Web Development Project - A web application that provides a solution to a real-world problem. Involves completing a set of tasks and meeting specific guidelines to get the job done.
  • Minimum Viable Product (MVP) - Usually the first iteration of a product without extra features that fulfils the requirements and proves that the solution to the program works.
  • Version Control - Allows us to track and manages changes to our codebase so we can make changes or revert back to an earlier version.
  • Repository (Repo) - A folder in which all files and their version histories are stored.
  • Branch - A workspace in which you can make changes that won't effect the live site.
  • Markdown (.md) - Light-weight formatting language used to write and style in GitHub that converts plain text to GitHub code.
  • Commit - A saved record of a change made to a file within the repo. Happens when adding, editing, or deleting a file within a repo inside a branch.
  • Pull Request (PR) - The way to ask for changes made to a comparison branch to be merged into another branch (typically main branch) that also allows for multiple users to see, discuss, and review the work being done.
  • Merge - After a pull request is approved, the commit will be pulled in (or merged) from one branch to another and then deployed on the live site.
  • Issues - How work is tracked when using Git. Allow users to report new tasks and content fixes, as well as allows users to track progress on a project board from beginning to end of a specific project.
  • Federalist - A platform that securely deploys a website from a GitHub repo in minutes & lets users preview proposed and published changes.
  • README Files - Used to pass along useful info to anyone who visits the repo on GitHub. They're titled README because they're the file that any visitor to the repo should read before they start exploring or contributing to it.
  • GitHub Issues - Task management tool offered by GitHub which gives developers and teams an easy way to keep track of key tasks and progresses.
  • Blocker - A particular problem you're stuck on that is preventing you moving forward.
  • Bottlenecks - Point in development process that is slowing down the rest of production.
  • Feature - Functional part of the application or website that has value or fulfils a need in the business.
  • Local Environment - Workspace where each developer does their work independently on their own computer.
  • Integrated Development Environment (IDE) - Type of software used by developers that combines many of the features needed by developers to do their work in a single interface, or GUI.
  • Graphical User Interface (GUI) - Icons and menus that allow the user to navigate the system by using mouse, touchpad, or assistive technology for more user friendly access instead of interacting with the system using CLI.
  • VS Code - Visual Studio Code used to write, view, and edit our code.
  • -m Flag - Associates a message with our commit
